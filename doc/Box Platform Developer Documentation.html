Get Started with the Box API

The quickstart guide below will take you through all the steps necessary to start accessing your own Box account through the API.

Authenticating Users

Instant Mode: For an additional, easier way to create content in a user’s account, please see the tokens endpoint.
  1. Get a new ticket by calling the following method with your API key: 
    GET https://www.box.com/api/1.0/rest?action=get_ticket&api_key={your api key}

    You’ll receive an XML response like this:

    <response>
  <status>get_ticket_ok</status>
  <ticket>bxquuv025araaaaaze2n4md9zef95e8</ticket>
</response>
  2. Direct the user to: 
    https://www.box.com/api/1.0/auth/{your ticket}

  3. The user is responsible for entering their credentials at this page.
  4. Upon completion, you can retrieve the auth_token either by:
    • Setting a redirect URL in your application settings that Box will post the auth_token to.
    • calling the following method with corresponding response 
      GET https://www.box.com/api/1.0/rest?action=get_auth_token&api_key={your api key}&ticket={your ticket}
      <response>
  <status>get_auth_token_ok</status>
  <auth_token>9byo5bg8d2o3otp0voji0ej0v49bqcmo</auth_token>
  <user>
    <login>stas@itscript.com</login>
    <email>stas@itscript.com</email>
    <access_id>453</access_id>
    <user_id>453</user_id>
    <space_amount>2147483648</space_amount>
    <space_used>1024</space_used>
    <max_upload_size>26214400</max_upload_size>
  </user>
</response>

Accessing User’s Content

Once you’ve successfully authenticated your user, you’ll want to be able to see what’s in their Box account. You can do this by simply getting the information about their root folder, which for every user is identified by ’0′. Using cURL, we would do this like so:

curl https://www.box.com/api/2.0/folders/FOLDER_ID \
-H "Authorization: BoxAuth api_key=API_KEY&auth_token=AUTH_TOKEN"

Uploading and Downloading

Now that we’re able to see what’s in the user’s account, let’s try to add some new files through the API. We need to do this as a multipart form upload, which looks like this in cURL:

curl https://www.box.com/api/2.0/files/data \
-H "Authorization: BoxAuth api_key=API_KEY&auth_token=AUTH_TOKEN" \
-F filename=@FILE_NAME \
-F folder_id=FOLDER_ID
Alert: Don’t forget the “@” sign when indicating the filename. This tells cURL to read data from the file.

Upon success, you’ll receive a response that should look like this:

{
    "type":"file",
    "id":"69446236",
    "name":"Book1.xls",
    "description":"",
    "size":5632,
    "path":"\/1234\/Book1.xls",
    "path_id":"\/6453368\/69446236",
    "created_at":"2012-04-18T11:15:11-07:00",
    "modified_at":"2012-04-18T16:08:00-07:00",
    "shared_link_enabled":false,
    "shared_link":"",
    "sha1":"cab902e7ec24a36588377554f2765eab57ecf86c",
    "modified_by":
    {
        "type":"user",
        "id":"1320308",
        "name":"Donald Glover"
    },
    "owned_by":
    {
        "type":"user",
        "id":"1320308",
        "name":"Donald Glover"
    },
    "parent":
    {
        "type":"folder",
        "id":"6453368",
        "name":"1234"
    }
}

Since we now have a file_id available, let’s try downloading that same file just to see how the process works. This is accomplished through a simple GET request to a /files resource like so:

curl https://www.box.com/api/2.0/files/FILE_ID/data?version=10849 \
-H "Authorization: BoxAuth api_key=API_KEY&auth_token=AUTH_TOKEN"

The response to this request will simply be the complete data of the file itself.
